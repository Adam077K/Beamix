Adam wants product brainstorming done as "board meetings" with multiple subagents providing different expert perspectives (SMB persona, marketing team, pricing expert, UX expert, etc.). He values:

- **Critical views, not agreement.** "Be more critical, like a board meeting." He wants diverse opinions that challenge assumptions.
- **Real cost validation.** He caught that AI cost estimates were 2-3x too low because they missed search/grounding fees. Always validate cost estimates against real production data and official API pricing including surcharges.
- **Bottom-up pricing.** Don't set prices then justify them — understand the system costs first, then price. "If we don't understand what systems we are gonna use, we can't price it correctly."
- **Don't keep features from old product automatically.** "I don't want to keep anything from the previous product besides the UI/UX." Rethink everything from first principles.
- **Value must be tangible.** A $49 tier with 5 credits was rejected because "they won't see the value they are looking for."

**Why:** Adam is a hands-on technical founder who tests assumptions against real-world experience. He doesn't accept theoretical estimates at face value.

**How to apply:** For any product/pricing/architecture decision, deploy 3-5 parallel researcher agents with different critical lenses. Always include a "devil's advocate" perspective. Validate all cost estimates against official API pricing pages including hidden fees (search surcharges, grounding fees, per-request fees, platform markups).
